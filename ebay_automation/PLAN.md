# Sam's Collectibles - eBay Listing Automation

## Overview

This automation pipeline helps Sam's Collectibles go from "photos on disk + HTML descriptions" to "ready-to-upload eBay CSV" with minimal manual work. It handles inventory tracking, competitive pricing via sold listings, and bulk CSV generation compatible with eBay's Seller Hub Reports uploader.

## How It Works

### Phase 1: Inventory Tracker (`inventory_tracker.py`)

Answers the question: "What do I have ready to list vs. what's already on eBay?"

**Inputs:**
- `ebay_uploads/` folder (scanned for product photo folders)
- `ebay_descriptions/` folder (scanned for completed HTML files)
- Active listings report CSV (downloaded from Seller Hub > Reports > Downloads)

**Process:**
1. Scans `ebay_uploads/` to find all product folders that contain photos
2. Scans `ebay_descriptions/` to find all completed HTML description files
3. Reads the active listings report CSV to get current eBay inventory
4. Cross-references everything using a fuzzy name-matching system
5. Categorizes each product as: READY (photos + HTML, not on eBay), NEEDS_DESCRIPTION (photos only), ALREADY_LISTED, or PHOTOS_ONLY

**Output:**
- Console status report
- `data/inventory_status.json` - machine-readable status for the CSV generator

**Usage:**
```bash
python inventory_tracker.py
python inventory_tracker.py --active-report path/to/active-listings.csv
```

### Phase 2: Sold Price Lookup (`sold_price_lookup.py`)

Answers the question: "What's the highest price this item has sold for recently?"

**Inputs:**
- Product keywords (from HTML title or manual entry)
- eBay Finding API credentials

**Process:**
1. Takes product title keywords
2. Calls eBay Finding API `findCompletedItems`
3. Filters for SOLD items only (not just completed/unsold)
4. Extracts all sold prices
5. Returns highest sold price as recommended Buy It Now price

**Output:**
- Recommended price per item
- `data/price_cache.json` - cached prices to avoid redundant API calls

**Setup Required:**
- Free eBay Developer account at https://developer.ebay.com
- App ID (Client ID) from the developer dashboard
- Store the App ID in `config.py` or as environment variable `EBAY_APP_ID`

**Usage:**
```bash
python sold_price_lookup.py "1993 SkyBox Star Trek Deep Space Nine sealed box"
python sold_price_lookup.py --all  # Price all items in inventory_status.json
```

**API Limits:**
- 5,000 findCompletedItems calls per day (more than enough)
- Results go back ~90 days of completed listings

### Phase 3: CSV Generator (`csv_generator.py`)

Answers the question: "Give me a CSV I can upload to eBay right now."

**Inputs:**
- `data/inventory_status.json` (from Phase 1)
- `data/price_cache.json` (from Phase 2)
- HTML description files (from `ebay_descriptions/`)
- eBay CSV template (from `templates/`)

**Process:**
1. Reads inventory status to find READY items
2. For each item, reads the HTML description file
3. Looks up cached pricing (or uses a default/manual price)
4. Fills in the 161-column eBay CSV template with:
   - Action: Add
   - Category: mapped from product type (261035 for Sealed Boxes, etc.)
   - Title: extracted from HTML header
   - Description: full HTML content
   - ConditionID: mapped from product type
   - StartPrice: from sold price lookup
   - Format: FixedPrice
   - Quantity: 1 (default)
   - Location: from config
   - Shipping: from config defaults
   - Returns: from config defaults
   - Item Specifics (C: fields): extracted from product metadata
5. PicURL is left blank for now (added manually in Seller Hub after uploading as drafts)

**Output:**
- `output/ebay_upload_YYYY-MM-DD.csv` - ready to upload to Seller Hub > Reports > Uploads

**Usage:**
```bash
python csv_generator.py
python csv_generator.py --items "ds9-series-1,master-series"  # specific items only
python csv_generator.py --drafts  # upload as drafts (no PicURL required)
```

## Workflow (Day-to-Day Usage)

### Adding a New Product

1. Take photos, put them in the appropriate `ebay_uploads/` subfolder
2. Paste the product text into Cowork to generate the HTML description (ebay-html-builder skill)
3. Run `python inventory_tracker.py` to confirm it shows as READY
4. Run `python sold_price_lookup.py "product keywords"` to get pricing
5. Run `python csv_generator.py` to generate the upload CSV
6. Upload CSV to Seller Hub > Reports > Uploads (as drafts)
7. Open each draft in Seller Hub, add photos, review, and publish

### Bulk Listing Session

1. Download a fresh active listings report from Seller Hub > Reports > Downloads
2. Save it to `ebay_automation/data/`
3. Run `python inventory_tracker.py --active-report data/latest-report.csv`
4. Run `python sold_price_lookup.py --all` to price everything
5. Run `python csv_generator.py` to generate the CSV
6. Upload and publish

## Future Enhancements (When Website Is Live)

- **Image Hosting**: Upload photos to website, fill PicURL with real URLs, skip the draft step entirely
- **Full Automation Skill**: Cowork skill to run the entire pipeline with a single command
- **Scheduled Price Updates**: Periodic re-pricing based on latest sold data
- **Inventory Sync**: Auto-detect when photos are added and prompt for listing
- **eBay Trading API**: Direct listing creation without CSV (bypasses Seller Hub entirely)

## Configuration

All settings are centralized in `config.py`:
- File paths (uploads, descriptions, CSVs, output)
- eBay defaults (location, shipping, returns, store policies)
- Category mappings
- Condition mappings
- API credentials (or environment variable references)

## Dependencies

See `requirements.txt`. Core dependencies:
- `ebaysdk` - eBay API SDK for Python (Finding API)
- `beautifulsoup4` - HTML parsing for extracting titles/metadata
- `thefuzz` - Fuzzy string matching for inventory comparison
- Standard library: `csv`, `json`, `os`, `pathlib`, `datetime`, `argparse`
